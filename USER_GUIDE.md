## 1. Що таке SecureShare

SecureShare — це програма для **одноразової безпечної передачі файлів** між двома користувачами через інтернет.

### Ключові особливості

- 🔒 **Наскрізне шифрування** — ваші файли зашифровані від початку до кінця. Навіть сервер-посередник не бачить вмісту
- 🔑 **Верифікація з'єднання** — візуальний код для підтвердження, що ви з'єднані саме з потрібною людиною
- ✅ **Перевірка цілісності** — після передачі файл автоматично перевіряється (SHA-256)
- 📦 **Один файл (.exe)** — нічого встановлювати, просто запустіть
- 🌐 **Працює через NAT** — не потрібна «біла» IP-адреса чи налаштування роутера

### Як це працює (коротко)

```
Відправник                     Сервер                          Отримувач
    │                            │                                 │
    │── Підключення ────────────►│◄──────────── Підключення ──────│
    │                            │                                 │
    │── Обмін ключами ──────────►│── Пересилає зашифровано ──────►│
    │                            │                                 │
    │── Верифікація ◄───────────────────────────► Верифікація ────│
    │                            │                                 │
    │══ Зашифрований файл ══════►│══ Пересилає байти ════════════►│
    │                            │                                 │
    │── SHA-256 ◄───────────────────────────────── SHA-256 ───────│
    │                            │                                 │
```

Сервер виступає тільки як «поштове відділення» — пересилає зашифровані дані, не маючи ключів для їх розшифрування.

## 3. Надсилання файлу

### Покрокова інструкція

**Крок 1.** Запустіть `SecureShare.exe`

**Крок 2.** Переконайтесь, що обрана вкладка **«📤 Надіслати»** (вгорі)

**Крок 3.** Натисніть кнопку **«📂 Огляд»** та оберіть файл для передачі

- Після вибору ви побачите назву файлу та його розмір
- Якщо файл більше 5 ГБ — з'явиться жовте попередження (див. [Обмеження](#8-обмеження))

**Крок 4.** Натисніть кнопку **«📤 Надіслати»**

- Програма згенерує **код сесії** — унікальний код, наприклад: `a7f3-bc21`
- Цей код з'явиться на екрані

**Крок 5.** Передайте код сесії отримувачу

- Натисніть кнопку **📋** поруч із кодом щоб скопіювати його в буфер обміну
- Відправте код отримувачу через месенджер, телефоном, будь-яким зручним способом

**Крок 6.** Зачекайте, поки отримувач підключиться

- Статус зміниться на: «🟡 Очікування партнера…»

**Крок 7.** Підтвердіть верифікацію (див. [Верифікація](#5-верифікація-зєднання))

**Крок 8.** Передача розпочнеться автоматично

- Ви побачите прогрес-бар із відсотком, швидкістю та залишковим часом
- Статус: «🟢 Передача…»

**Крок 9.** Дочекайтесь повідомлення

- `🎉 Файл передано та перевірено ✓` — успіх!
- Статус зміниться на «🟢 Завершено ✓»

## 4. Отримання файлу

### Покрокова інструкція

**Крок 1.** Запустіть `SecureShare.exe`

**Крок 2.** Перейдіть на вкладку **«📥 Отримати»** (вгорі)

**Крок 3.** Введіть **код сесії**, який дав вам відправник

- Ви можете ввести код вручну (формат: `xxxx-xxxx`)
- Або натисніть кнопку **«📋 Вставити код»** — код автоматично вставиться з буфера обміну

**Крок 4.** Натисніть **«📂 Папка»** та оберіть куди зберегти файл

**Крок 5.** Натисніть кнопку **«📥 Отримати»**

- Програма підключиться до сервера та знайде відправника
- Статус: «🟡 Підключення…» → «🟡 Очікування відправника…»

**Крок 6.** Підтвердіть верифікацію (див. [Верифікація](#5-верифікація-зєднання))

**Крок 7.** Файл почне завантажуватися

- Ви побачите назву файлу, прогрес, швидкість та залишковий час

**Крок 8.** Дочекайтесь завершення

- `✅ Збережено: назва_файлу.ext (X.X МБ/с)` — файл збережено
- Файл з'явиться у вибраній папці

## 5. Верифікація з'єднання

### Що це таке

Після того як відправник і отримувач з'єднались, програма показує обом **код верифікації** — 8-символьний код у форматі `E555-EB8B`.

Цей код однаковий для обох сторін **тільки якщо** з'єднання безпечне.

### Навіщо це потрібно

Верифікація захищає від **атаки «людина посередині» (MITM)** — ситуації, коли зловмисник перехоплює з'єднання та підмінює шифрувальні ключі. Якщо така атака відбувається, коди будуть **різними**.

### Що робити

1. **З'являється діалогове вікно** з великим кодом верифікації
2. **Зателефонуйте** або **зверніться** до партнера та порівняйте коди:

   | Ситуація | Що бачите | Дія |
   |----------|-----------|-----|
   | ✅ Коди однакові | `E555-EB8B` = `E555-EB8B` | Натисніть **«Так, коди збігаються»** |
   | ❌ Коди різні | `E555-EB8B` ≠ `A123-B456` | Натисніть **«Ні»** — з'єднання буде розірвано |

3. **Обидва** учасники мають підтвердити — якщо хоча б один натисне «Ні», передача скасовується

> **Важливо:** Не ігноруйте верифікацію! Це єдиний спосіб переконатися, що ваше з'єднання не перехоплене.

## 7. Діагностика

Перед першою передачею рекомендуємо натиснути кнопку **«🔍 Діагностика»** у верхній частині вікна.

### Тести діагностики

Програма виконує 5 перевірок:

| # | Тест | Що перевіряє | Якщо не пройшов |
|---|------|-------------|-----------------|
| 1 | 🌐 Інтернет | TCP з'єднання з `1.1.1.1:443` | Немає доступу до інтернету |
| 2 | 🔗 DNS | Розв'язання доменного імені сервера | Проблема з DNS (спробуйте `8.8.8.8`) |
| 3 | 🔒 TLS/SSL | Валідність SSL-сертифіката сервера | Сертифікат прострочений або мережа блокує |
| 4 | ⚡ WebSocket | Повне WSS з'єднання з сервером | Файрвол блокує WebSocket (порт 443) |
| 5 | 📊 Латентність | Затримка (пінг) до сервера | Показує час відгуку в мілісекундах |

### Результати

- ✅ **Зелений** — тест пройшов
- ❌ **Червоний** — тест не пройшов (з описом помилки)
- Якщо всі 5 тестів зелені — програма готова до роботи

## 8. Обмеження

| Обмеження | Деталі | Рекомендація |
|-----------|--------|-------------|
| **Максимум 5 ГБ за сесію** | Це ліміт сервера. Після 5 ГБ з'єднання буде розірвано | Для більших файлів: розбийте архіватором (7-Zip, WinRAR) на частини до 4 ГБ |
| **Один файл за сесію** | Протокол передає один файл | Для кількох файлів: запакуйте в ZIP/RAR архів |
| **Потрібен інтернет** | На обох пристроях одночасно | Мобільний інтернет теж підходить |
| **Код сесії одноразовий** | Після використання — недійсний | Для нової передачі — нова сесія |
| **Windows та Linux** | macOS не підтримується офіційно | macOS: запуск з вихідного коду (Python 3.11+) |

### Про ліміт 5 ГБ

Ліміт встановлений на сервері для запобігання зловживанням. Він діє на **загальний обсяг зашифрованих даних** за одну сесію. Оскільки шифрування додає ~3-5% накладних витрат:

| Розмір файлу | Зашифрований обсяг | Пройде? |
|-------------|-------------------|---------|
| 1 ГБ | ~1.04 ГБ | ✅ |
| 4.5 ГБ | ~4.7 ГБ | ✅ |
| 4.8 ГБ | ~5.0 ГБ | ⚠️ На межі |
| 5 ГБ+ | >5.2 ГБ | ❌ Буде перервано |

**Рекомендація:** для гарантованого результату не перевищуйте **4.5 ГБ** на один файл.

## 9. Безпека

### Що захищено

| Що | Як захищено |
|----|------------|
| **Вміст файлу** | Шифрування AES-256-GCM (наскрізне) — сервер не бачить вмісту |
| **Назва та розмір файлу** | Зашифровані в контрольних повідомленнях |
| **Ключі шифрування** | Обмін ключами X25519 — ключі ніколи не передаються відкрито |
| **Від підміни з'єднання** | Код верифікації — захист від MITM-атаки |
| **Від підробки даних** | AES-GCM автентифікує кожен блок + SHA-256 для всього файлу |
| **Від перехоплення** | WSS (TLS 1.2+) — шифрований канал до сервера |

### Що НЕ захищено

| Що | Пояснення |
|----|-----------|
| **Код сесії** | Передається вами вручну — захищайте його! Не публікуйте в відкритих чатах |
| **IP-адреса** | Сервер бачить вашу IP-адресу (потрібно для роботи) |
| **Факт передачі** | Сервер знає, що відбулась передача (але не вміст) |
| **Метадані на вашому комп'ютері** | Файл зберігається незашифрованим на диску отримувача |

### Поради з безпеки

1. **Завжди перевіряйте код верифікації** — зателефонуйте або попросіть партнера назвати код
2. **Передавайте код сесії захищеним каналом** — Signal, Telegram (секретний чат), телефон
3. **Не публікуйте код сесії** у відкритих чатах, на форумах, у соціальних мережах
4. **Використовуйте останню версію** програми — оновлення можуть містити виправлення безпеки

## 10. Журнал подій (логи)

### Де знайти логи

Програма автоматично зберігає логи у файл:

```
%APPDATA%\SecureShare\secureshare.log
```

Зазвичай це:
```
C:\Users\<ваше_імʼя>\AppData\Roaming\SecureShare\secureshare.log
```

### Як отримати логи з програми

| Кнопка | Дія |
|--------|-----|
| **📋 Лог** | Копіює весь журнал з вікна програми у буфер обміну |
| **💾 Зберегти лог** | Зберігає журнал у обраний текстовий файл (.txt) |

### Коли знадобляться логи

- Якщо передача не працює — логи допоможуть зрозуміти причину
- Якщо потрібна допомога — надішліть лог адміністратору

## 11. Вирішення проблем

### Загальна діагностика

Перед пошуком конкретної проблеми:
1. Натисніть **«🔍 Діагностика»** → переконайтесь, що всі 5 тестів зелені
2. Перевірте, що у обох учасників є інтернет
3. Переконайтесь, що обидва використовують **останню версію** програми

---

### «Не вдалось підключитися до relay»

**Причина:** Програма не може з'єднатися з сервером.

**Рішення:**
1. Перевірте інтернет — відкрийте будь-який сайт у браузері
2. Запустіть діагностику (🔍) — подивіться який тест не пройшов
3. Якщо DNS не працює — спробуйте змінити DNS на `8.8.8.8` (Google) або `1.1.1.1` (Cloudflare)
4. Якщо корпоративна мережа — можливо, файрвол блокує WebSocket на порту 443. Зверніться до адміністратора
5. Спробуйте через мобільний інтернет (точка доступу з телефону)

---

### «Таймаут очікування партнера»

**Причина:** Відправник чекав отримувача більше 5 хвилин, або навпаки.

**Рішення:**
1. Переконайтесь, що обидва натиснули кнопку майже одночасно (різниця — до 5 хвилин)
2. Перевірте, що код сесії введений **правильно** (без зайвих пробілів)
3. Використовуйте кнопку **📋** для копіювання та **📋 Вставити код** для вставки — це виключає помилки
4. Спробуйте ще раз — натисніть «Надіслати» знову для нового коду

---

### «Невірний формат обміну ключами» / «Помилка розшифрування ключа»

**Причина:** Код сесії введений неправильно, або технічна проблема з'єднання.

**Рішення:**
1. Перевірте, що код **точно** збігається в обох програмах
2. Створіть новий код (натисніть «Надіслати» знову)
3. Переконайтесь, що обидва учасники мають **однакову версію** програми

---

### «Несумісна версія партнера»

**Причина:** Партнер використовує застарілу версію програми.

**Рішення:**
1. Обидва учасники мають завантажити останню версію
2. Завантажити: [secureshare-relay.duckdns.org/download/SecureShare.zip](https://secureshare-relay.duckdns.org/download/SecureShare.zip)

### «Хеш не збігається»

**Причина:** Файл був пошкоджений під час передачі.

**Рішення:**
1. Спробуйте надіслати файл ще раз
2. Якщо повторюється — перевірте стабільність інтернету
3. Спробуйте менший файл для тестування

---

### Передача дуже повільна

**Причина:** Швидкість обмежена найповільнішим з'єднанням (відправника або отримувача), а також пропускною здатністю сервера.

**Рішення:**
1. Переконайтесь, що обидва мають стабільний інтернет
2. Уникайте завантажень / стрімінгу під час передачі
3. При можливості — використовуйте дротове з'єднання замість Wi-Fi
4. Нормальна швидкість: **1–10 МБ/с** (залежить від інтернету)

---

### Передача перервалась на середині

**Причина:** Втрата інтернет-з'єднання або таймаут.

**Що відбувається автоматично (v3.2):**
- 🔄 Програма автоматично спробує **перепідключитись** (до 5 спроб: 5с → 10с → 20с → 40с → 60с)
- ✅ При успішному перепідключенні — верифікація пропускається автоматично
- 📦 Передача **продовжується з місця зупинки** (не з початку)

**Якщо автоматичне перепідключення не вдалось:**
1. Програма зберігає прогрес у `.resume` файл (автоматично)
2. Дочекайтесь стабільного з'єднання
3. Відправник обирає **той самий файл** і натискає «Надіслати» (новий код)
4. Отримувач вводить новий код
5. Передача продовжиться з місця зупинки (resume)
6. Прогрес зберігається **7 днів**

### Windows SmartScreen блокує запуск

2. Натисніть **«Run anyway»** (Все одно запустити)

### Браузер блокує завантаження .zip

3. Firefox: зазвичай завантажує без проблем

### Антивірус видаляє або блокує .exe

**Рішення:**

## 12. Часті запитання (FAQ)

### Чи безпечна ця програма?

**Так.** SecureShare використовує ті ж алгоритми шифрування, що й Signal, WhatsApp, та інші сучасні месенджери:
- X25519 для обміну ключами
- AES-256-GCM для шифрування
- SHA-256 для перевірки цілісності

Сервер працює як «сліпий посередник» — він пересилає зашифровані байти, не маючи ключів для їх розшифрування.

### Чи бачить сервер мої файли?

**Ні.** Шифрування відбувається на вашому комп'ютері ще до відправки. Сервер бачить лише зашифрований потік даних, який він не може розшифрувати.

### Що буде, якщо сервер зламають?

Зловмисник отримає доступ тільки до зашифрованих даних. Без ключів шифрування (які є тільки у відправника та отримувача) розшифрувати їх **неможливо**.

### Чи потрібна реєстрація?

**Ні.** Ні акаунтів, ні паролів, ні email. Просто запустіть програму та передайте файл.

### Чи можна передати кілька файлів?

За одну сесію — один файл. Для кількох файлів — запакуйте їх в один архів (ZIP, RAR, 7z) і передайте його.

### Чи можна передати файл більше 5 ГБ?

Ліміт сервера — 5 ГБ за сесію. Для більших файлів:
1. Розбийте файл на частини за допомогою 7-Zip або WinRAR (до 4.5 ГБ кожна)
2. Передайте кожну частину окремою сесією
3. На стороні отримувача — зберіть файл назад

### Як оновити програму?

Починаючи з версії 3.3, програма **автоматично перевіряє оновлення** при кожному запуску. Якщо є нова версія — з'явиться діалог з кнопкою «Оновити зараз». Оновлення завантажується, перевіряється (SHA-256) та встановлюється автоматично.

Також можна перевірити вручну — натисніть кнопку **🔄** у верхній частині вікна.

Якщо у вас старіша версія — просто завантажте нову і замініть старий файл.

### Чи працює через VPN?

**Так.** SecureShare використовує стандартний WebSocket через порт 443 (HTTPS) — працює через більшість VPN та проксі-серверів.

### Чи працює з мобільного інтернету?

**Так.** Будь-яке інтернет-з'єднання підходить. Але для великих файлів рекомендується стабільне Wi-Fi або дротове з'єднання.

### Де зберігаються отримані файли?

У папці, яку ви обрали при натисканні кнопки «📂 Папка» на вкладці «Отримати». За замовчуванням це робочий стіл або папка «Завантаження».
